Incident Command System is a standardized, on-scene, all hazard management framework used by agencies to organize emergency responses. 

### Key Principles
- **Unified Command:** Multiple agencies work together under a common set of objectives without losing individual authority.
- **Chain of Command**: An orderly line of authority within the response organization.
- **Unity of Command**: each responder reports to only one supervisor.
- **Modularity**: The system expands or contracts based on the size and complexity of the incident.
- **Span of Control**: Supervisors ideally manage 3–7 personnel, with 5 being optimal.

### Functional Modules
ICS is organized into five major functional areas:
- **Command**
    - Incident Commander (IC)
    - Establishes objectives
    - Overall responsibility for incident management
- **Operations**
    - Carries out tactical objectives    
    - Direct management of responders in the field
- **Planning**
    - Collects and evaluates information
    - Develops action plans    
    - Tracks resources and situation status 
- **Logistics**
    - Provides resources and support
    - Supplies, equipment, personnel, rehab, communications
- **Finance / Administration**
    - Tracks costs
    - Contracts, claims, timekeeping, documentation

ICS is implemented during:
- HazMat events
- Natural Disasters
- [[Mass Causality Incidents]]
